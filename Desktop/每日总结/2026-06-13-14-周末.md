### Windows Claude Code 环境修复

**问题**：Claude Code 启动报错 — "requires either Git for Windows or PowerShell"

**根因**：Git 安装在 `C:\Git\`（非默认路径），`bash.exe` 不在 PATH 中。

**解决**：
1. 设环境变量 `CLAUDE_CODE_GIT_BASH_PATH=C:\Git\bin\bash.exe`
2. Node.js 加入 User PATH

> 教训：Windows 上装软件不走默认路径会留一堆坑。

花时间了解了 Claude Code 的可扩展能力：

- **Skill** = 专业化的工作流脚本，像给 AI 装专业工具

- **Memory** = 跨会话持久化记忆

### 二、Token 经济学认知

学习了 Skills 的 Token 开销：

| 场景 | Token 消耗 | 建议 |
|------|:--:|------|
| 启动加载 14 个 Skill | ~2,500 | 固定开销 |
| 激活单个 Skill | <5,000 | 按需激活 |
| 同时激活 | 1-3 个最佳 | 多了浪费 |
| 复杂任务用全套 | 省 ~14% | 避免返工 |
| 简单任务用全套 | 浪费 | 小修小补别开 |

**决策**：只保留 brainstorming + using-superpowers 启用，其余 12 个默认禁用，需要时手动开启。避免"杀鸡用牛刀"。

## 📚 学到的核心知识

### Claude Code 工作原理

```
用户输入 → Claude 分析 → 匹配 Skill → 执行工作流 → 输出结果
                ↑
            Memory 提供上下文
            Hook 自动触发
            Skill 提供专业流程
```

### 关键认知

1. **Skills 不是插件，是工作流** — 它们决定 AI 如何思考、分步、验证，不是简单加功能
2. **brainstorming 是最值得开的 Skill** — 苏格拉底式反问，能把模糊需求逼出清晰定义
3. **Token 预算意识** — 简单任务 + 复杂 Skill = 浪费。匹配任务复杂度选工具
4. **Memory 是 AI 的长期记忆** — 写一次，永久生效。比每次都口头重复强
5. **跨电脑同步是刚需** — git push/pull memory + skills = 两台电脑无缝切换

### RESIONE 网站开发铁律

从之前踩坑中总结的 7 条规则：

1. **禁止 JSX 硬编码文案** — 全走 `useTranslations()`
2. **三语 JSON 必须同步** — en/zh/ja 键值完整覆盖
3. **PDF 只用 reportlab** — xhtml2pdf 中文乱码，weasyprint Windows 装不上
4. **next-intl 必须 setRequestLocale** — 否则回退英文
5. **产品数据以官方产品册为准** — PDF > 官网 > 第三方
6. **EYE skill 交叉验证** — 中英文 PDF 分别 OCR 对照
7. **部署走 Gitee** — GitHub 被墙，Vercel 直接 deploy

## 💡 最大收获

> **AI 工具不是越花哨越好，是越匹配越好。**
>
> 装 20 个 Skill 不等于生产力 ×20。关键是：
> - 知道什么任务用什么工具
> - 知道什么时候不开工具更高效
> - 把经验写成 Memory/SOP，让 AI 记住不再犯
>
> 这个认知比装任何 Skill 都值钱。
