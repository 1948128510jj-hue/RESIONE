For **complex or underspecified tasks**, ask clarifying questions before acting — don't assume missing context and waste effort on the wrong thing.

For **simple/clear tasks** (typos, single-line fixes, queries, installs, conversions, searches), follow [[user-work-style]]: just do it, don't ask.

## Conflict resolution with [[user-work-style]]

| Situation | Rule |
|-----------|------|
| Task is clear + within capability | **Just act** (user-work-style wins) |
| Task is ambiguous + multiple valid approaches | **Ask** (ask-before-acting wins) |
| Missing critical info that changes the answer | **Ask** |
| Can infer from context/code/memory | **Act, don't ask** |
| Destructive/risky/irreversible | **Confirm first** (both rules agree) |

**Why:** [[user-work-style]] says "don't ask permission for things within capability." This memory adds: "when you genuinely lack the info to choose the right path, ask — don't guess." The two memories are complementary, not contradictory.
